## 2. ロゴアニメーション仕様

### 表現
- `typo.` + 点滅カーソル `|`（細い縦線、2px幅）
- カーソルはCSSアニメーションで制御（530ms間隔、ease-in-out）
- フォントウェイトは400〜500（軽すぎず重すぎず）
- コマンドライン的すぎない柔らかさを保つ

### アニメーションシーケンス（約4秒）

| 時間 | 動作 | 意図 |
|------|------|------|
| 0.0s〜0.8s | 空の状態でカーソルが点滅 | 「これから何かが始まる」期待感 |
| 0.8s〜1.4s | `t → ty → typ → type` 入力 | 自然なタイピングリズム |
| 1.4s〜1.8s | 停止（カーソル点滅継続） | 「あれ、間違えた？」の間 |
| 1.8s〜2.2s | バックスペースで `type → typ → ty → t` | 打ち間違い＝typoの体現 |
| 2.2s〜2.5s | 停止 | 考え直す間 |
| 2.5s〜3.0s | `t → ty → typ → typo` 再入力 | 正解への到達 |
| 3.0s〜3.2s | `.` 入力（カーソル軽く跳ね） | 句点＝完結の強調 |
| 3.2s〜3.5s | カーソルフェードアウト | ロゴとして確定 |
| 3.5s〜4.0s | タグラインフェードイン | ブランドメッセージの着地 |
| 4.0s〜4.3s | CTAボタンフェードイン | 行動喚起 |
| 5.0s | スクロールヒント表示 | 下層コンテンツへの誘導 |

### カーソル跳ねアニメーション詳細
- `.` 入力時にカーソルが2px上に跳ね、戻る
- duration: 150ms
- easing: cubic-bezier(0.34, 1.56, 0.64, 1)（軽いバウンス）

## 3. LP構成

### 3.1 ヒーローセクション（100vh）

#### レイアウト
```
┌─────────────────────────────────────┐
│                                     │
│                                     │
│              typo.|                 │  ← 垂直中央よりやや上（40vh位置）
│                                     │
│       タイピングに、余白を。          │  ← ロゴから32px下
│                                     │
│                                     │
│            [ はじめる ]              │  ← タグラインから48px下（任意）
│                                     │
│               ∨                     │  ← スクロールヒント
└─────────────────────────────────────┘
```

#### 仕様
- ロゴサイズ: clamp(48px, 10vw, 96px)
- タグライン: clamp(14px, 2.5vw, 18px)
- 背景: #fafafa（純白より目に優しい）
- ロゴ位置: 垂直中央よりやや上（視覚的重心の調整）
- スクロールヒント: 5.0s で表示開始、上下にゆっくり動く（Y方向 4px、2s周期）

## 4. 技術スタック

### アーキテクチャ方針
- **フロントエンドのみで完結** — バックエンド不要
- **静的ファイルのみ** — HTML/CSS/JS を出力し、任意の静的ホスティングで動作
- **外部API依存なし** — すべてのコンテンツはコード内に定義
- **軽量** — 不要なフレームワークを排除し、最小構成で実現

### コア技術

| カテゴリ | 技術 | バージョン | 理由 |
|----------|------|-----------|------|
| ライブラリ | React | 19 | コンポーネントベースUI |
| ビルドツール | Vite | 6.x | 高速ビルド、軽量、設定シンプル |
| 言語 | TypeScript | 5.x | 型安全性 |
| スタイリング | Tailwind CSS | 4.x | ユーティリティファースト、ビルド時最適化 |

### 追加ライブラリ（必要に応じて導入）

| ライブラリ | 用途 | 導入タイミング |
|-----------|------|---------------|
| Framer Motion | ロゴアニメーション、フェードイン | Phase 2 |
| clsx | 条件付きクラス名結合 | 随時 |

### 使用しないもの（意図的に除外）

| 技術 | 除外理由 |
|------|----------|
| Next.js / Remix / Astro | LPには過剰、静的ファイルで十分 |
| データベース | フロントエンド完結のため不要 |
| 認証 | LPに不要 |
| CMS | コンテンツ量が少なく、コード内管理で十分 |
| CSS-in-JS (styled-components等) | Tailwindで統一 |
| 状態管理ライブラリ (Redux等) | ページ単体で完結、グローバル状態不要 |
| SSR/SSG フレームワーク | 単一ページのため不要 |

### ビルド・デプロイ

| 項目 | 選択 |
|------|------|
| ビルドコマンド | `npm run build` → `dist/` に出力 |
| 出力形式 | 静的ファイル（HTML, CSS, JS） |
| ホスティング | GitHub Pages / Cloudflare Pages / Netlify / Vercel / S3 など任意 |
| CI/CD | GitHub Actions（任意） |

### フォント

| フォント | 用途 | 読み込み方法 |
|----------|------|-------------|
| Geist Sans | 本文、UI | Google Fonts または セルフホスト |
| Geist Mono | ロゴ、コード表現 | Google Fonts または セルフホスト |

### ディレクトリ構成（予定）

## 6. アクセシビリティ

### 必須対応
- `prefers-reduced-motion: reduce` 時はアニメーションをスキップし、完成状態を即時表示
- ロゴにaria-label="typo. タイピングに、余白を。"を付与
- CTAボタンはフォーカス可視化（outline）
- カラーコントラスト比 4.5:1 以上を確保

### 推奨対応
- キーボードナビゲーション対応
- スクリーンリーダーでの動作確認
